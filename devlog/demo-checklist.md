# AstralFox 面试演示 Checklist

> 用途: 面试/展示前逐条验证, 确保不翻车
> 每条 = 操作步骤 + 预期结果 + 变通方案

---

## 准备阶段 (开始前 5 分钟)

- [ ] 1. 确认 Python BFF 已启动: `cd backend && python main.py`, 看到 "Uvicorn running on"
- [ ] 2. 确认 Unity 已打开 AstralFox 场景, Console 无红色 Error
- [ ] 3. 确认麦克风权限已授予, Windows 设置中麦克风未静音
- [ ] 4. 备选方案: 如果 BFF 连不上, 按 F12 进入 Mock 模式 (WakeWordDetector mock)

---

## 演示流程 (2 分钟)

### 阶段 1: 启动 (30 秒)

- [ ] 5. 双击 exe 或 Unity Play → 狐狸出现在桌面右下角
- [ ] 预期: 透明窗口 + Live2D 模型渲染 + 默认 Idle 动画(眨眼/呼吸)
- [ ] 变通: 如果没有透明窗口, 检查 TransparentWindow._disableTransparency 是否为 false
- [ ] 6. 拖拽狐狸到桌面中间 → 验证拖拽物理(弹性跟随 + 松手弹跳)
- [ ] 预期: 狐狸跟随鼠标有弹性延迟, 松手后有小幅弹跳

### 阶段 2: 语音对话 (60 秒)

- [ ] 7. 说 "小星小星" (唤醒词) → 狐狸切换到 Listening 状态
- [ ] 预期: 狐狸耳朵竖起, 身体前倾, 眼睛注视用户
- [ ] 变通: 如果语音唤醒失败, 按 F12 模拟唤醒
- [ ] 8. 说 "你好, 今天天气怎么样" → 等待回复
- [ ] 预期: 气泡显示 "正在理解你说的话…" → "星尘正在思考…" → 狐狸张嘴说话
- [ ] 变通: 如果 ASR 不可用, Mock 模式会随机返回假转录
- [ ] 9. 观察 streaming 效果: token 逐字出现在气泡中, 情绪标签触发表情切换
- [ ] 预期: 回复过程中狐狸表情随情绪标签变化 (happy→微笑, sad→低落)

### 阶段 3: 错误恢复 (20 秒)

- [ ] 10. 在狐狸说话时, 关闭 Python BFF (Ctrl+C) → 模拟断连
- [ ] 预期: 气泡显示错误提示(BFF 连接失败), 狐狸回到 Idle 不卡死
- [ ] 11. 重启 BFF → 狐狸自动重连 → 气泡显示 "正在连接 AI 服务…"
- [ ] 预期: 恢复后可以说 "小星小星" 继续对话

### 阶段 4: 展示技术亮点 (10 秒)

- [ ] 12. 在设置中展示 PAD 情绪面板 (如果有 UI)
- [ ] 预期: 显示 P/A/D 三个数值, 点击狐狸后数值上升
- [ ] 13. 展示 Live2D 参数驱动的 12 种空闲行为 (等 10-30 秒观察)
- [ ] 预期: 自动触发 Scratch/Stretch/Sneeze/LookAround 等随机行为

## 已知问题 & 面试应答

| 问题 | 面试中被问到时的回答 |
|------|-------------------|
| 离线 AI 为什么没开启 | "FunASR+Qwen7B 需要 16GB+ 内存, 当前演示用云端 BFF。离线方案的技术调研已完成, 详见 devlog。" |
| 为什么有两个主题色 | "BEM 迁移进行中, globals.css 和 global.scss 正在合并。当前统一使用暖色主题。" |
| 测试覆盖率多少 | "核心路径(VoiceManager 状态机 + DataStore 加密)有单元测试, 整体覆盖率约 15%。下一步补 UI 测试。" |
| 最难的 bug 是什么 | "Environment.TickCount 49.7 天溢出导致空闲检测失效。无符号减法一行修复, 但找 root cause 花了两天。" |
