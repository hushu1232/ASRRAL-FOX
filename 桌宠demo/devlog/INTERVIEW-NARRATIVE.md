## 故事 2: "你做过的最难的 bug"

### TickCount 49.7 天溢出

**问题**: 用户反馈狐狸在没人碰电脑时频繁跳出回访问候。

**调查**: 加了日志发现空闲检测的空闲时间是负数。`Environment.TickCount` 是 int32，Windows 上每 24.9 天溢出回零。空闲检测公式 `TickCount - lastInputTime` 在溢出后得到的是巨大的负数除以 1000，变成极大值——系统以为用户离开了几百万秒。

**修复**: 改 uint 无符号减法，无符号类型在溢出时自动回绕，差值永为正。

**教训**: 永远不要假设时间 API 的返回值不会溢出。Unity 的 `Time.time` 也有类似问题（float 精度在 33 小时后退化），但影响较小。

## 故事 3: "你最大的工程决策失误"

### 先建基础设施，后做核心体验

我在 Prisma ORM 上建了 19 个数据模型，用 Next.js 16 做了 30 个前端组件，用 K8s + Terraform + Grafana 搭了一套生产级部署流水线。

但我的角色模型是从 GitHub 上找的一个版权不确定的占位品。

回头看：应该先花两周搞定角色美术——找画师画立绘、AI 管线分离图层、验证动画参数映射。然后才考虑要不要做支付系统、创作者市场、A/B 实验这些当前用户数为零的功能。

但也不是完全没有价值。面试桌上我能解释为什么我做这些选择，以及我从中学会了什么。一个从没犯过错的高级工程师，大概率从没做过真正复杂的系统。
