Unity-AIChat 表现力层深度解析

基于源码级分析 (38 个 C# 文件) — 仅关注用户可见/可听/可交互层面

1. 角色视觉呈现

做了什么
模型类型⚠️ 3D 模型 (SkinnedMeshRenderer)，非 Live2D。自带 Genshin 胡桃 .mesh 文件 (12 个材质)
窗口形态⛔ 无透明窗口 — 标准 Unity Windows 窗口，角色在灰色背景的 3D 场景中，无桌面融合
渲染质量一般 — 使用 Unity 内置管线 + 简单材质。无后处理、无抗锯齿设置
怎么做的
Mesh: Assets/AIChatTookit/Mesh/胡桃/ (12 个 .mat 材质文件)
Shader: Built-in Standard, 无自定义 Shader
Camera: 默认 Main Camera, 无 RenderTexture 或透明通道

效果评价：差。完全没有桌面宠物应有的透明窗口和桌面融合感。这更接近一个 "3D 角色查看器" 而非桌宠。
对星尘狐借鉴：不值得参考。我们已有 Live2D + Unity 透明窗口，领先于此项目。

2. 动画与表情系统

维度有/无实现方式效果借鉴
眨眼✅BlinkController.cs: SkinnedMeshRenderer.SetBlendShapeWeight, 每 3s 触发, 0.2s 动画, Lerp 插值一般 — 频率固定, 无随机变化, 像机械定时器需改进后复用
口型同步✅AudioMouthController.cs: AudioSource.GetOutputData(512样本) → 均方根振幅 → SmoothDamp → BlendShape一般 — 响度驱动, 不是音素驱动。嘴巴只是"开合"可参考, 但应升级为音素级
待机动画❌无 — 角色完全静止, 无呼吸/微动/随机动作差 — 像雕塑⛔ 不必参考
情绪表情❌无 — 仅 Animator.SetInteger("state", 0/1/2) 切换三个状态差 — 仅 idle/thinking/speaking 三种⛔ 不必参考
LLM 动作指令❌未实现 — LLM 回复纯文本, 无 [wave]/[nod] 等标记解析不存在⛔ 不必参考
移动动画❌未实现 — 角色不可移动, 无路径寻找不存在⛔ 不必参考

关键发现：Animator 状态机 (0=idle, 1=thinking, 2=speaking) 是项目中唯一可借鉴的动画管理方式。但 AstralFox 的 Live2D 有更丰富的参数系统。

3. 语音交互体验

维度有/无实现方式效果借鉴
语音输入✅VoiceInputs.cs: Microphone.Start() 录制 (5s, 16kHz) → WAV 字节数组 → HTTP POST 到 Whisper API基本可用 — 需手动按住按钮录音, 不支持自动断句UI 交互模式可参考
倾听反馈⚠️ 仅文字按钮文字变化: 按下→"正在录音...", 松开→"正在识别..."简陋 — 无动画/波形/气泡⛔ 需完全重做
TTS 合成✅GPTSoVITSTextToSpeech.cs: 参考音频 base64 + 文本 → HTTP POST → 返回 WAV 路径 → UnityWebRequest 加载播放可用 — 支持参考音色克隆, 但每次 HTTP 往返有延迟API 调用范式可参考
状态指示⚠️ 仅文字Text 组件显示 "正在思考..." → Animator.SetInteger("state",1)简陋 — 用户感知不清晰⛔ 需完全重做
语音打断❌未实现不存在⛔ 需自行实现

4. 物理交互反馈

维度有/无说明借鉴
点击角色❌无 Raycast 检测, 无点击反馈 — 所有交互通过 UGUI 按钮⛔
拖拽角色❌未实现 — 窗口不可拖动角色⛔
鼠标追踪❌无视线跟随/身体转向⛔
碰撞检测❌无物理/碰撞体⛔

表现力维度全部缺失。Unity-AIChat 是一个纯功能导向的项目，完全没有物理交互层的表现力设计。

5. UI 与视觉元素

维度有/无实现效果借鉴
对话气泡❌使用 Unity UGUI Text 组件 + ScrollRect 面板, 非桌面气泡像聊天软件, 不像桌宠⛔
打字机效果✅ChatSample.StartTypeWords(): IEnumerator 逐字显示, 0.2s/字, 可中断可用 — 简单的逐字动画值得借鉴
粒子特效❌无 — 项目无任何 ParticleSystem 引用不存在⛔
设置界面✅UGUI InputField + Dropdown, 用于配置 API URL 和参数功能可用但不美观⛔
聊天历史✅List<string> → ChatPrefab 实例化显示, 左(用户)/右(AI)布局, ScrollRect 自动滚到底部基本可用⛔ 已有 Web 版

维度有/无说明借鉴

结论： Unity-AIChat 在表现力层面几乎无借鉴价值。它的价值在于 LLM/TTS/STT 的插件化架构设计（策略模式 + ScriptableObject），而非用户可见的表现力。AstralFox 在 Live2D 渲染、透明窗口、角色动画方面已大幅领先。

唯一值得直接借鉴的：打字机效果 (IEnumerator 逐字显示) — 代码结构清晰，可直接移植到 Unity 客户端。
