4. 核心竞争力分析

🟢 独特优势

  • 全离线原生 — 唯一同时支持本地 LLM + ASR + TTS + 声音克隆的桌宠产品
  • 创作者经济 — 唯一具有完整形象市场的桌宠平台 (模型/人格卡/语音包/动画交易)
  • 企业级基础设施 — 唯一的 K8s/Helm/ArgoCD/Terraform 部署、Prometheus 监控、SLO 管理的桌宠项目
  • Web+Unity 双端 — BFF 层复用认证/存储/数据库，Web 端管理 + Unity 端运行
  • 多模型支持 — Live2D + VRM + DragonBones，竞品仅支持单一格式

🟡 待追赶

  • 跨平台支持 — 竞品 WindowPet 支持 Win/Mac/Linux，Codex Pets 支持 macOS，AstralFox 仅 Windows
  • 安装包体积 — 完整 Unity+AI 引擎约 2-5GB，WindowPet < 10MB
  • macOS 市场 — Codex Pets (macOS 独占) 拥有大量开发者用户

🔴 竞争威胁

  • Steam 桌宠爆发 — "AI Desktop Pet" 和 "你的AI桌宠" 在 Steam 快速增长，拥有成熟分发渠道
  • AI Agent 桌宠融合 — Codex Pets 模式将桌宠与 AI 开发工具深度绑定，开辟新品类
  • 硬件化趋势 — EMO/Loona/Moflin 提供物理触感交互，纯软件桌宠有被替代风险

5. 战略建议

短期 (1-3 月)：巩固核心优势
  1. 完成 OpenAI/Azure 残留清理后的全离线验证
  2. 降低安装包体积 — 考虑 modular AI 引擎按需下载
  3. Steam 上架 — 利用现有 Unity 构建发布到 Steam 获取分发渠道
中期 (3-6 月)：拓展平台 + 生态
  1. macOS 移植 — 使用 Unity macOS build target
  2. AI Agent 集成 — 参考 Codex Pets 模式，将桌宠与 Claude Code 等 AI 编程工具联动
  3. 创建者 SDK — 让第三方开发者创建和销售 Live2D 模型/人格卡/语音包
长期 (6-12 月)：平台化 + 硬件探索
  1. 硬件伴侣 — 低成本 ESP32 蓝牙设备 (参考 Claude Buddy M5StickC)
  2. 跨平台统一 — 考虑 Tauri 重写轻量版前端，降低资源占用
  3. 开放 API — 让第三方 AI 应用接入 AstralFox 形象系统
