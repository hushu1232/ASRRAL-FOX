2. 性能优化（P1 — 速度感知）

React-Admin 启发的性能模式 + 现有 Web Vitals 监控
模式 react-admin 做法 本项目适配 预期收益
乐观更新 useMutation + undoable SWR optimisticData + Toast 撤销 感知延迟 → 0ms
路由预取 <link prefetch> 已有 router.prefetch (hover+visible) 导航延迟 → <100ms
代码分割 lazy + Suspense per route next/dynamic (图表、3D、编辑器) 首屏 JS -30%
图片优化 <Image /> lazy next/image + 缩略图 + blurDataURL LCP -40%
数据缓存 react-query cache SWR dedup + Redis 服务端缓存 重复请求 → 0
虚拟列表 <Datagrid rowClick> 大数据集（>100条）使用虚拟滚动 列表渲染 FPS 稳定
SSR 优先 SSR + hydration 关键页面 RSC + Stream + Suspense FCP -50%
📊 Web Vitals 监控（已有）
PerformanceMonitor 已收集 LCP/CLS/INP/FCP，通过 web-vitals → /api/metrics → Prometheus → Grafana。
优化前基线：LCP ~2.8s, CLS ~0.12, INP ~180ms。目标：LCP < 1.5s, CLS < 0.05, INP < 100ms。
